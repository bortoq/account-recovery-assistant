# Feedback Guide

Thanks for trying Account Recovery Assistant. This is a controlled alpha, so the most valuable contribution is clear feedback about whether the plan is understandable, safe, and useful.

## Do Not Share Sensitive Data

Please do **not** send or paste:

- passwords;
- backup codes;
- SMS codes;
- authenticator codes;
- identity document scans or numbers;
- payment card numbers;
- private account identifiers you do not want public.

Use placeholders such as `my Gmail account`, `old recovery email`, or `business tenant`.

## Best Feedback To Send

If you tried the tool, please include:

1. Which incident did you test?
   - Google MFA loss
   - Apple trusted device / phone loss
   - Meta hacked account
   - Microsoft personal account recovery
   - Microsoft 365 admin / tenant lockout
   - Other
2. Did the **Next Best Action** make sense?
3. Which checklist/evidence item was confusing?
4. Did the official link work?
5. Did the plan help you avoid a bad recovery action?
6. What scenario should be added next?
7. Would you prefer:
   - self-serve local tool;
   - hosted web version;
   - human-assisted / concierge guidance;
   - business/MSP recovery readiness workflow?


## Where To Send Feedback

- Open a GitHub issue using one of the templates.
- Comment on the alpha tester issue if one is pinned.
- If you contact the maintainer privately, still avoid sending passwords, codes, IDs, or private account data.

## What Success Looks Like

Stars are appreciated, but the alpha is mainly looking for:

- real recovery scenarios to improve;
- confusing plan sections;
- stale or broken official links;
- safety concerns;
- evidence that the plan helped someone take a better first step.
