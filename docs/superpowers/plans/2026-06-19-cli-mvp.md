# Account Recovery Assistant CLI MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a tested Python CLI that turns a recovery situation JSON file into a safe account recovery plan.

**Architecture:** A small Python package exposes pure functions for loading playbooks, classifying scenarios, and generating plans. The CLI reads input JSON and prints a JSON plan. Playbooks are stored in `data/recovery_playbooks.json`.

**Tech Stack:** Python standard library, `pytest`, JSON files, module CLI via `python -m account_recovery_assistant`.

---

## File Structure

- Create `pyproject.toml`: package metadata and pytest config.
- Create `src/account_recovery_assistant/__init__.py`: public exports.
- Create `src/account_recovery_assistant/__main__.py`: CLI entrypoint.
- Create `src/account_recovery_assistant/playbooks.py`: playbook loading and validation.
- Create `src/account_recovery_assistant/planner.py`: scenario classification and plan generation.
- Create `data/recovery_playbooks.json`: three initial safe playbooks.
- Create `examples/lost_mfa.json`, `examples/changed_phone.json`, `examples/suspicious_lock.json`: demo inputs.
- Create `tests/test_planner.py`: behavior tests.
- Create `tests/test_cli.py`: CLI JSON output test.
